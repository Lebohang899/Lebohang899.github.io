without going to the actual store itself. This website is aimed to allow users to explore menu items,
find information quickly, and submit enquiries when necessary. The aim is to provide an enjoyable and

## Website Goals and Objectives
- Goal 1 - Effective online service
Customers are provided with easily accessible information about KFC products. Making use of the the products
page allows the customer to view menu items, prices, sizes and quantity options. Products are organised into
simpler sections so that they can be searchable and hence easily findable by customers.

- Goal 2 - User-friendly and responsive website
The website consists of navigation features thereby allowing the customer to travel between the different web
pages easily. Responsive ddesign techniques were implemented so that the website adjusts to the conditions of
multiple different devices. This ensures consistency throughout so that the user can have an enjoyable experience
regardless of the device.

- Goal 3 - Promote KFC's brand image
The website promotes interaction through the use of enquiry and contact forms, product search functionality as well
as interactive JavaScript features. These features are designed to make the website more engaging. This plays a role
in stregthening the comminication between the business and the customer, thereby generating a positive brand image.

## Part 2 Details
Regarding this part of the project I made use of CSS to adjust the desktop layout, typography, colour palette and responsiveness. CSS was designed to make the KFC website appear more minimalistic, easy to comprehend and more user-friendly(Ducket,2021). I made use of a navigation bar which can move easily between home, products, specials, enquiry, contact and services page. 

On the desktop the website uses a more wider layout with the product cards arranged neatly in rows. Each card is spaced in a manner in which the page does not seem crowded in order to improve its readability(Robbin, 2021). I had made use of KFC's brand identity for the colour palette. Red is used for important areas such as buttons, and highlights. Black is used for headings from within the page and the navigation bar when hovering over, whereas white is utilized to keep the page clean and readable. 

The website is designed to be responsive, which means that it can adjust to different screen sizes. For example on phones and tablets the layout becomes smaller and easier to sscroll through. In addition the products stack neatly, and the images resize properly, furthermore the navigation bar remains simple to use. This makes the device user-friendly across multiple devices(Freeman, 2023).

## Part 3 Details
This part of the project focuses more on JavaScript features, SOE work, form validation and processing, and website deployment. An essential feauture that was implemented using JavaScript was the product search functionality. This allows users to find menu items quicker by typing in a few keywords. JavaScript was also made use of to improve navigation, generate interactive elements and also to supply a smoother browsing experience throughout different pages to make it easier for the user. 

I had made use of Search Engine Optimization (SOE) methods in order to enhance the website's visibility and accessibility. For this part I had made use of titles, meta description, and relevant keywords in order to guide search engines. In addition suitable heading structures and descriptive page names were used to organise the information logically.

Form validation was implemented to make sure that users entered accurate results and also completed the information before submitting the form. Required information is firstly checked by the engine upon submission. Through validation users are prevented from submitting missing information or wrong information. This ensures that the collected information is reliable(Farrell, 2023). 

Enquiry and contact forms are designed so that the users information are captured correctly and so that information can be delivered effectively. Once the user completes the form, JavaScript processes the information and checks that all the conditions of the required information has been met. If it turns out the the information submitted is valid, then a confirmation message will appear thus indicating to the user that they have provided valid details.

Regarding the deployment of the website, a structured folder system containing HTML, CSS, JavaScript, image resources, README and sitemap was followed. It is essential that all web pages are linked correctly to ensure effortless navigation after publishing. Furthermore the project was uploaded via GitHub for version control and project management, which allows for changes to be tracked throughout the development of the website. In conclusion the website can therefore be hosted through GitHub pages or other platform(Farrell, 2023).
